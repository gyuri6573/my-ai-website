复盘心得 / Lesson Learned
我卡在哪一步了？
- YouTube 网页端登录验证： YouTube 网页端用新注册谷歌账号登录时遇到验证问题，暂未完全解决；
- 豆包局限性：豆包无法总结部分平台链接内容（B 站、小红书、非公开抖音视频），仅能通过录音记录文字稿总结，操作繁琐且效果有限；
- Notion 渲染报错：在尝试用 AI 生成复杂流程图时，遇到了 Mermaid 语法对特殊字符（如圆括号、HTML标签）不兼容的问题，导致渲染失败。
- Notion高级功能：虽然掌握了基本功能，但还没有完全熟悉其深度应用，未来需要进一步探索Notion的自动化和数据库功能。

你是如何解决你的问题的？
- 豆包工具使用问题：发现链接总结功能的局限性后，尝试用豆包 “录音记录 + 文字稿总结” 功能替代，虽操作繁琐，但能满足基础视频内容总结需求；同时明确了豆包支持的有效链接类型（公开抖音视频）；
- Notion渲染问题：通过观察报错信息，让AI进行二次修改后成功渲染。
- Notion更多功能问题：我将逐步积累使用经验，未来会继续学习如何通过数据库和自动化功能提升Notion的工作流管理能力。

这次教学卡片你的收获是什么？
- 工具层面：系统梳理了 AI 各领域主流工具，完成工具注册与初步试用，明确了不同工具的适用场景；
- 技能层面：掌握了提示词工程核心原则；了解了 Markdown/JSON 格式的本质、用途及区别；
- 效率的降维打击：亲身体验了“对话 -> 代码 -> 图表”的工作流，相比传统手工绘图，效率提升。
- 提示词心法：在需求模糊时，不急着下指令，而是先赋予 AI “需求分析师”的角色，让它引导我明确方向。
- 身份的数字化：明白了在 AI 时代，明确的角色定位（Role）是获取高质量答案的钥匙。

01
任务画像 / Context
周期：1.3 - 1.7 | 目标：进阶实操图、视、音等多模态创作。
02
行动清单 / Action
- 将女儿的线条涂鸦，利用Gemini/豆包润色为成熟的插画作品；
- 用海艺ai 生成IP“蓝白化工机器人”，并利用“图生图+固定风格提示词”，生成了风格统一的海报；
- 用可灵/海螺首尾帧技术制作了类似苹果广告的产品演示动画；
- 用豆包生成了自己的Q版形象；
- 用剪映克隆了自己的音色，配合AI生成的机器人，制作了短片；
03
成果展示 / Show Time
04
复盘心得 / Lesson Learned
我卡在哪一步了？
- 文生图和视频时，很难达到自己想要的效果。目前的解决方案只能是笨办法（不断换提示词、换工具试错）。深刻体会到提示词工程对行业经验要求极高，不是随便写写就能成的。

你是如何解决你的问题的？
- 人机协作：利用豆包共享屏幕功能，让AI充当“手把手教我的老师”，实时解决剪映操作问题。这比自己搜教程快得多。
- 化整为零：视频生成放弃“一镜到底”，采用分段生成 + 后期剪辑策略，用剪辑软件的确定性去弥补AI生成的不确定性。
- 结构化喂料：在图生图时，不直接给参考图，而是先让AI“读图”并输出 JSON 配置单，把“感性的参考图”变成了“理性的数据参数”再喂给AI，成功率显著提升。

这次教学卡片你的收获是什么？
- 提示词是门槛：AIGC不是零门槛的，它对使用者的“行业认知”和“语言精确度”要求极高。
- 流程优于工具：无论是“JSON提取”还是“分段生成”，本质上都是在优化工作流 (Workflow)。工具不够完美时，好的流程可以弥补缺陷。
- MVP思维：完成比完美重要。一个跑通的粗糙视频，价值远高于一个永远停留在构想中的完美电影。

复盘心得 / Lesson Learned
我卡在哪一步了？
- Github验证：GitHub 注册时“Visual Puzzle”（视觉拼图）连续两次尝试均失败；
- 插件不可用：GitHub汉化插件安装后无法运行；
- gemini code assist在vs code无法登录

你是如何解决你的问题的？
- Github验证问题：切换验证方式，用“Audio Puzzle”（语音验证）。事实证明，当一种 AI 验证方式过于死板时，尝试多模态验证（听觉）是高效的备选方案；
- 浏览器插件问题：浏览器底层机制问题，汉化插件失效是因为浏览器“无痕模式”自动拦截了所有扩展插件，切换回“正常模式”后插件生效；
- gemini code assist无法登录问题：更改vs code代理配置

这次教学卡片你的收获是什么？
- 工具环境的一致性：意识到开发环境的稳定性，很多时候“工具坏了”其实是“系统设置不兼容”，保持正常浏览环境是配置开发工具的前提。
